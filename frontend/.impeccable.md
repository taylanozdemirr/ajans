# LUXE AGENCY — Frontend Design System

## Design Context

### Users
Three audiences across three surfaces:
- **Public visitors (vitrin / `/`)** — clients, brands, casting/event organizers browsing the agency's roster of models ("manken"). They scan portraits quickly and reach out via WhatsApp. Context: browsing a curated showcase, often on mobile, expecting a high-end, gallery-like impression.
- **Agencies / companies (`/dashboard` → company panel)** — partner agencies who log in to publish and manage their model listings against a quota ("ilan hakkı"). Context: focused admin work, want clarity and speed.
- **Platform admin (`/dashboard` → admin panel)** — internal operator managing companies and their listing limits.

Language: **Turkish**.

### Brand Personality
Three words: **refined · editorial · confident**. Think a fashion lookbook and a gold-foiled matte business card — not nightclub gold, not corporate SaaS. The interface should feel like a boutique talent agency: quiet luxury, generous space, strong typography, photography as the hero.

Canonical brand name: **LUXE AGENCY** (was inconsistently "TMÖ AGENCY" / "TMÖ ADMIN" / "LUXE AGENCY" — now unified everywhere).

### Aesthetic Direction
- **Direction**: Luxury / refined — an evolution of the existing gold-on-black, executed with precision.
- **Theme**: Both light and dark are supported via a class-based toggle persisted to `localStorage` (default **dark**). The **public vitrin is always dark** (showcase). Panels (login, company, admin) respect the user's toggle.
- **Anti-references**: generic SaaS dashboards, neon/cyber gold, AI-slop (side-stripe cards, gradient text, glass-everything, identical card grids, big rounded icons over every heading).

### Typography
- **Display** (`font-display`): **Bodoni Moda** — high-contrast Didone for the wordmark, hero, model names, and large headings. Fashion-magazine masthead energy. Never use below ~20px; never for body.
- **Body / UI** (`font-sans`): **Hanken Grotesk** — clean, slightly warm grotesque for all UI, forms, tables, captions.
- Loaded via Google Fonts in `Layout.astro` (the `@fontsource-variable/geist` dependency is unused / not installed; Geist was never actually applied).

### Color (OKLCH tokens)
Single source of truth in `src/index.css`, consumed via Tailwind as `oklch(var(--token) / <alpha-value>)` so `/opacity` utilities work.
- **Gold (`--primary`)**: one consistent antique-champagne gold, hue ~84. Replaces the old split between the `hsl(43 100% 50%)` token and the hardcoded `#D4AF37`.
- Neutrals are **tinted toward the gold hue** (very low chroma) for cohesion. No pure `#000` / `#fff`.
- 60 / 30 / 10: deep warm-black (or off-white) surfaces, muted secondary text/borders, gold reserved as a rare accent.

### Design Principles
1. **One source of truth** — tokens, fonts, brand, and panel chrome are defined once and reused. No per-page drift.
2. **Photography is the hero** — on the vitrin, type and chrome stay restrained so the models carry the page.
3. **Gold is rare** — accent, not wallpaper. Hierarchy comes from type weight/size and space, not color.
4. **Quiet motion** — one orchestrated entrance per view, ease-out; the marquee is the only ambient motion. Respect `prefers-reduced-motion`.
5. **Professional defaults** — no native `confirm()`, no broken brand, consistent spacing rhythm, accessible contrast in both themes.
